# Sahajeevanam Puram – Financial Model Assumptions (V1)

## Status

This is a concept-stage financial assumptions and scenario model. It is not a DPR, lender model, or investment memorandum. Its purpose is to make the current pitch more grounded by connecting the development envelope, funding mix, revenue logic, scenario cases, and Phase 0 ask.

## 1. Development Envelope

### Concept-stage total development estimate

- Approximate total:
  `INR 5,200 crore`

This should be treated as an illustrative planning envelope derived from the current design thesis, not as finalized project costing.

## 2. Suggested Capital Stack for External Pitching

Present the funding stack as a target blend, not as closed fact:

- State / SPV support:
  `30%`
- PPP / strategic infrastructure partners:
  `25%`
- Green and social bonds:
  `20%`
- Mission-linked institutional / development finance:
  `15%`
- CSR / philanthropy / catalytic grants:
  `10%`

## 4. Scenario Cases

The workbook now includes three operating cases:

- `Conservative`:
  slower workforce ramp, softer mentorship realization, delayed IP commercialization, and a `10%` Phase 0 cost uplift
- `Base`:
  midpoint case used for the main dashboard and narrative
- `Upside`:
  stronger workforce utilization, better mentorship conversion, faster IP commercialization, and no Phase 0 inflation uplift

These cases are designed to support safe external conversations. They are not forecasts of committed revenue.

## 5. Sensitivity Views

The workbook includes two quick sensitivity grids:

1. `Year 7 Revenue Sensitivity`
   Digital workforce realization versus IP licensing realization
2. `Year 7 Revenue / Phase 0 Ask`
   Digital workforce realization versus Phase 0 budget inflation

These help answer the most common meeting-room questions:

- what happens if workforce execution is slower than expected,
- what happens if IP monetization takes longer,
- and how much the revenue story depends on Phase 0 cost discipline.

## 9. How to Pitch This Safely

- Do not claim bankability yet.
- Do not claim guaranteed returns.
- Do not present scenario outputs as contracted revenues.
- Do present the workbook as a concept-stage tool used to justify Phase 0 authorization and deeper diligence.

## 10. Suggested Language for Pitches

"The current financial model is a concept-stage assumptions and scenario workbook. It is intended to justify Phase 0 authorization and feasibility work, not to substitute for DPR-grade costing, land diligence, or investor underwriting."
